# 树洞小程序 V1 需求文档（仅含 6 个附加功能）

> 本文档是 V1 **唯一执行基线**。范围外需求（耳语、时空回声、双风格高级动效等）暂不进入本期开发。

## 1. V1 目标与范围

### 1.1 目标
在现有小程序基础上，优先上线“缓解心情”核心闭环：
- 写下情绪 → 释放/安慰 → 记录情绪 → 感官放松。

### 1.2 本期仅实现 6 个功能
1. 粉碎
2. 解忧盲盒
3. 情绪调色盘
4. 呼吸岛屿
5. 雨落字里
6. 潮汐白噪音

---

## 2. 功能定义（V1）

## 2.1 粉碎（发布页）

### 用户价值
把负面内容“写出来并释放掉”，不进入公开流。

### 交互要求
- 在发布页输入内容后点击“粉碎”。
- 触发轻量粒子消散动画（V1 允许简化为 CSS 粒子/淡出）。
- 震动反馈一次：`wx.vibrateShort()`。
- 清空输入框并提示：`已粉碎`。

### 数据规则
- 不创建帖子，不进入首页流。

---

## 2.2 解忧盲盒（我的页面）

### 用户价值
快速获得一条安慰文案，形成即时情绪支持。

### 交互要求
- 点击盲盒卡片，执行翻转动画。
- 从本地文案池随机抽取 1 条文案展示。
- 支持“再来一句”（重新抽取）。

### V1 约束
- 先使用前端本地数组，不依赖后端接口。

---

## 2.3 情绪调色盘（我的页面）

### 用户价值
每天用颜色记录情绪，看到情绪变化轨迹。

### 交互要求
- 每天首次进入可选择“今日心情色”（12 选 1）。
- 在“我的页面”展示近 30 天色块。
- 点击色块可查看日期与对应主题名。

### 数据规则
- 本地存储：`moodCalendar`（日期、themeId、color）。

---

## 2.4 呼吸岛屿（首页 FAB）

### 用户价值
通过长按按钮进行呼吸节律引导，快速平复焦虑。

### 交互要求
- FAB 保持可拖动。
- 长按 1 秒进入呼吸态，按钮循环缩放（V1 用 4-4 或 4-7-8 简化版均可）。
- 呼吸态期间每个周期轻震动一次。
- 松手立即退出呼吸态并恢复正常样式。

---

## 2.5 雨落字里（首页）

### 用户价值
提供可开关的沉浸氛围，降低情绪噪音。

### 交互要求
- 首页提供“雨天模式”开关。
- 开启后在内容层上方渲染轻量雨滴动画（V1 可用 CSS/Canvas 简化）。
- 不阻塞页面滚动与点击。

### 性能要求
- 低端机可自动降级（减少粒子数量或关闭溅射）。

---

## 2.6 潮汐白噪音（全局）

### 用户价值
主题与听觉联动，增强沉浸感。

### 交互要求
- 每个主题绑定一个 `audioSrc`。
- 切换主题时执行淡入淡出。
- 首页提供播放/暂停控制。
- 用户上次播放状态和音量写入 Storage。

### 技术要求
- 使用全局 `InnerAudioContext`。

---

## 3. 技术实现边界（V1）

- 框架：微信小程序原生（WXML/WXSS/JS）。
- 不引入重型动画引擎；优先可维护、可降级实现。
- 所有新增状态优先本地存储，后端接口后续迭代。

---

## 4. 文件改造建议（按现项目）

- `app.js`
  - 统一白噪音播放控制、淡入淡出、播放状态缓存。
- `theme.config.js`
  - 补齐主题与音频字段映射（确保 12 个主题可取到 `audioSrc`）。
- `pages/home/index.js` / `index.wxml` / `index.wxss`
  - 呼吸岛屿、雨天模式开关与动画层。
- `pages/post/index.js` / `index.wxml` / `index.wxss`
  - 粉碎流程（动画 + 清空 + 提示）。
- `pages/profile/index.js` / `index.wxml` / `index.wxss`
  - 解忧盲盒与情绪调色盘（月历色块）。

---

## 5. 开发顺序（建议）

1. 先打通 **潮汐白噪音**（全局能力，其他功能会复用）
2. 实现 **粉碎**（发布页闭环）
3. 实现 **解忧盲盒**（我的页即时反馈）
4. 实现 **情绪调色盘**（本地存储 + 展示）
5. 实现 **呼吸岛屿**（FAB 长按节律）
6. 实现 **雨落字里**（可开关 + 性能降级）

---

## 6. 验收标准（V1）

1. 粉碎后内容不会出现在首页，输入框清空且有反馈。
2. 盲盒每次可稳定返回文案，翻转动画正常。
3. 调色盘可记录并展示近 30 天数据，重启后不丢失。
4. 呼吸岛屿长按可进入/退出节律动画，不影响拖动。
5. 雨天模式可开关，不卡主交互。
6. 主题切换时白噪音可平滑切换，播放状态可持久化。

---

## 7. V1 交互结构定稿（你确认：2）叠加 3）

## 7.1 主结构：时间轴疗愈型（方案2）

### 首页（Home）
- 顶部：推荐 / 附近 / 最新（保持现有 Tab）
- 中部：按时间段分组展示内容流（清晨 / 午后 / 深夜）
- 底部：仅保留核心入口（首页 / 我的 + 悬浮 FAB）

### 我的（Profile）
- 情绪调色盘作为核心模块（近 30 天）
- 解忧盲盒作为次核心模块（即时安慰反馈）

### 发布（Post）
- 沉浸输入为单一焦点
- 先做“发布 / 粉碎”双动作闭环

## 7.2 叠加样式策略：双风格切面型（方案3）

统一信息架构不变，仅切换视觉表达：

### 柔美组（主题 id 0-11）
- 大圆角（`40rpx+`）
- 弥散阴影
- 更大留白与更舒展行高
- 粉碎动效优先 `petal / bubble`

### 深邃组（主题 id 12-23）
- 小圆角（`12rpx~20rpx`）
- 描边优先（弱阴影）
- 更紧凑信息密度
- 粉碎动效优先 `shatter`

## 7.3 开发落地约束（V1）

1. 不改页面路由结构（仍使用 `home / post / profile`）。
2. 仅增加“时间段分组渲染”数据结构，不引入复杂推荐算法。
3. 风格切面通过主题字段直接驱动（`borderRadius / shadow / particleType / glassMode`）。
4. 不新增后端依赖，先用本地数据与本地状态完成闭环。

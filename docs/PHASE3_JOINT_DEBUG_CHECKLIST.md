# Phase 3 联调清单（Home 主链路）

> 目标：验证「主题 / 音频 / 雨落 / 呼吸 / 粉碎」在同一会话内互不干扰，满足 V1 可交付标准。

## 0. 测试环境

- 平台：微信开发者工具 + 至少 1 台真机（建议 Android 中端 + iOS）
- 账号状态：已清理缓存后首次启动、以及二次启动各执行一轮
- 基线：`pages/home/index` 默认进入首页

---

## 1. 主题 × 音频一致性

- [ ] 切换主题后，首页背景/卡片/按钮颜色同步变化
- [ ] 白噪音播放状态不因主题切换丢失
- [ ] 音量滑杆调整后，重启小程序仍保持上次值
- [ ] 播放中连续切换主题 5 次，无明显爆音/突兀断点
- [ ] 播放中手动改音量后再切主题，音量可平滑回归到用户设置值

---

## 2. 雨落字里（Home）

- [ ] 默认关闭（首次启动 `homeRainModeEnabled` 为 false）
- [ ] 开启后出现雨滴层，且**不阻塞**卡片滚动与按钮点击
- [ ] 关闭后雨滴层立即消失
- [ ] 退出并重进小程序后，雨天模式开关状态可恢复
- [ ] 低性能设备下雨滴数量明显减少（性能分级生效）

---

## 3. 呼吸岛屿（FAB）

- [ ] 短按 FAB：正常进入发布页
- [ ] 长按 FAB：进入呼吸引导（4-7-8），松手立即结束
- [ ] 长按结束后不会误触发跳转发布页（tap 抑制生效）
- [ ] 切后台/离开页面时自动停止引导

---

## 4. 粉碎链路（Home/Post）

- [ ] Home 卡片粉碎：动画 + 删除 + toast 正常
- [ ] Post 输入粉碎：动画 + 清空 + 返回首页正常
- [ ] 雨天模式开启时执行粉碎，不出现遮挡或点击失效
